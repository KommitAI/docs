---
title: "Introduction"
description: "Kommit is a project specification platform that gives AI coding tools persistent memory and context."
---

Kommit helps software teams define project specifications visually and share that context with AI coding tools through the [Model Context Protocol (MCP)](/mcp/overview).

You define your project on a visual canvas — features, data models, architecture decisions — and Kommit makes that knowledge available to tools like Claude Code, Cursor, and Windsurf as persistent memory.

## Key capabilities

<Columns cols={2}>
  <Card title="Canvas editor" icon="object-group" href="/features/canvas">
    Define project specs visually with an interactive node-based editor. Add features, data models, user flows, and connect them.
  </Card>
  <Card title="Memory system" icon="brain" href="/features/memory">
    Store decisions, learnings, and progress as searchable memories with vector embeddings. AI tools query these via MCP.
  </Card>
  <Card title="PRD generation" icon="file-lines" href="/features/prd-generation">
    Generate product requirements documents from your canvas specs using AI. Refine iteratively and export to Markdown or PDF.
  </Card>
  <Card title="Integrations" icon="plug" href="/features/integrations">
    Connect GitHub, Linear, Jira, Notion, Figma, and Slack to keep specs in sync with your workflow.
  </Card>
</Columns>
