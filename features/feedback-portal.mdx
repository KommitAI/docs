Kommit includes a feedback portal that lets you collect input from users, customers, or stakeholders without giving them access to your full project workspace.

## Public feedback pages

Each organization can create public feedback pages accessible via a URL like `getkommit.ai/f/{orgSlug}/{pageSlug}`. Visitors can submit feedback, vote on existing submissions, and leave comments — no account required.

## Submissions and voting

Feedback submissions support:

- **Title and description** for the feedback item
- **Voting** — users can upvote submissions to signal priority
- **Comments** — threaded discussion on each submission
- **Duplicate detection** — Kommit suggests merging similar submissions

## Admin moderation

From the admin dashboard, manage feedback with:

- **Status workflow** — move submissions through stages: new, under review, planned, in progress, shipped, declined
- **Inbox view** — conversation-style threading for each submission
- **Merging** — combine duplicate submissions and consolidate votes
- **Priority signals** — sort by vote count, recency, or engagement

## Connecting feedback to specs

When feedback is actionable, create canvas nodes directly from submissions. This links user requests to your project spec, closing the loop between what users ask for and what you build.
